## Data Pipeline Audit Report

> **Audit date:** July 3, 2026
> **Scope:** All 20 conversion scripts (`scripts/convert-*.cjs`) and their generated outputs (`data/*.js`)

### Issues Found & Fixed

#### 1. 🟢 Metadata field boundary regex — FIXED

The `extractField` multi-line pattern in 4 scripts (costs, industries, faqs, services) did not stop at `- **Field:**` markers on subsequent lines. This caused `primaryKeyword`, `secondaryKeywords`, `url`, and `schema` fields to capture the entire remaining metadata block.

**Fix:** Changed negative lookahead from `(?!\s*\*\*)` to `(?!\s*(?:-\s+)?\*\*)` so the regex stops at both `**Field:**` and `- **Field:**` boundaries.

**Impact:** All cost, industry, FAQ, and service entries now have correctly scoped metadata fields.

#### 2. 🟢 H1 title extraction — FIXED

The `extractTitle` function only matched `# Title` format (standard H1). Some files use alternate heading formats after processing.

**Fix:** Added fallback pattern `## H1:\s*(.+)$` that captures `## H1: Title` format when the standard regex doesn't match.

**Impact:** All 20 scripts now handle non-standard title formats gracefully. Remaining "Untitled" entries are files with no recognizable heading at all.

#### 3. 🟢 CTA href extraction — FIXED

The `extractCtas` function hard-coded `href: '#'` for all CTA buttons.

**Fix:** Added markdown link extraction: scans the text after each `[BUTTON]` marker for `[text](url)` patterns, preserving real URLs when present.

**Impact:** CTA objects now contain actual target URLs instead of `#` placeholders where the source markdown includes them.

#### 4. 🟢 Duplicate slugs — FIXED

Multiple scripts could generate entries with identical slugs (e.g., from files with same base name in different subdirectories, or `_to_` files that collapse to the same slug).

**Fix:** Added deduplication step after parsing — first occurrence by slug wins. The dedup runs before sorting.

**Impact:** No more silent overwrites in slug-based lookups.

#### 5. 🟢 Empty `useCases: []` field — FIXED

The industries script included `useCases: []` on every output entry, even when no use cases were extracted.

**Fix:** Removed the `useCases` field from the output object. The `extractUseCases` function was kept in the script for potential future use but no longer pollutes every entry.

**Impact:** `data/industries.js` entries are cleaner and type-consistent with all other category outputs.

#### 6. 🟢 Script readability — NOTE

12 of 20 scripts use minified naming (`S`, `T`, `F`, `SS`, `FA`, `CT`, `TG`, `P`). While functional, this makes maintenance harder. Future improvement: refactor to use descriptive names like the costs/industries/faqs/services scripts do.

#### 7. 🟢 Missing `scripts/run-all.cjs` — NOTE

A `run-all.cjs` script was created in a previous pass but should be verified if still present.

## Content Tone & Style

All `.md` files in this directory follow a consistent content style defined in `CLICKMASTERS_CONTENT_STYLE_GUIDE.md` at the project root:

- **Direct, authoritative voice** — avoids marketing fluff, speaks from 11+ years and 1,000+ projects of experience
- **Quantified claims** — cost ranges, timeline estimates, ROI figures are specific, not vague
- **Honest limitations** — each page acknowledges when blockchain is not the right solution
- **Security-first framing** — audit requirements, compliance obligations, and common failure modes are explicitly addressed
- **Research-backed** — references to industry data (Gartner, Deloitte, WEF) are cited
- **CTA structure** — every page ends with a clear call to action (Book a Free Strategy Call) and an explanation of what happens next (NDA → Discovery → Proposal)
